# Example Plans

This file contains three scenario-driven examples (S/M/L) to help validate
the High-Level Planning skill.

## Small (S) — Minor feature delivery

Input highlights
- Well-defined scope, single team, few dependencies.

Expected plan
- 2 phases: Design & Delivery
- estimated_size: S
- duration_range: { min_weeks: 2, max_weeks: 6 }

## Medium (M) — Data platform migration (example)

Input highlights
- Multiple data sources, some integration work, modest budget.

Expected plan
- 3 phases: Assessment; Target Architecture & Prioritization; Pilot & Handoff
- estimated_size: M
- duration_range: { min_weeks: 8, max_weeks: 20 }

## Large (L) — Enterprise transformation

Input highlights
- Cross-functional stakeholders, complex legacy systems, budget & timeline
  constraints.

Expected plan
- 4–5 phases covering discovery, architecture, incremental migration, scale-up,
  and organization handoff
- estimated_size: L
- duration_range: { min_weeks: 24, max_weeks: 52 }
